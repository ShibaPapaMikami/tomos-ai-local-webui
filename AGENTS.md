# TOMOS AGENTS

## 目的

TOMOS本体をAIエージェントが安全に編集・調査・検証するための作業ルールを定義する。

## 対象範囲

- TOMOS本体のコード、ドキュメント、テスト、リリース手順
- ローカルAI、PWA、契約書管理、教材パック、プラグイン導線
- Agent-Reach本体コードは対象外
- Agent-Reach受け入れ時は、TOMOS側アダプター、診断結果の読み取り、プラグイン表示、設定保存、既存TOMOS経路へのフォールバックだけを対象にする。

## 必須項目

- 作業前に対象範囲を確認する。
- 既存の未コミット変更を勝手に戻さない。
- 依存追加、削除、外部API実行、production変更は事前確認する。
- UI変更は `DESIGN.md` に従う。
- 会話・キャラクター変更は `VOICE.md` に従う。
- 記憶・保存・検索変更は `MEMORY.md` に従う。
- プラグイン仕様は `PLUGIN.md` に従う。
- 変更後は関連テスト、構文チェック、差分確認を行う。

## 任意項目

- 変更理由
- 影響範囲
- 手動確認手順
- ロールバック方法
- 未解決の確認事項

## 禁止事項

- Agent-Reach本体コードは変更しない。
- Agent-Reachのインストール処理、実行基盤、外部サービス認証、SNS連携、Cookie利用、外部書き込み処理は変更しない。
- Agent-Reach受け入れでは、外部アクセスを初期OFFまたはユーザー確認必須にする。
- TOMOS内データ参照は `PLUGIN.md` の `dataAccessScope` に明記された範囲だけに限定する。
- 承認済み設計に基づくTOMOS側アダプター、診断結果の読み取り、Web・YouTube・GitHub・RSSの既存TOMOS経路への接続は変更可能。
- SNS、Cookie、ログイン情報、外部書き込み連携は変更しない。

## Agent-Reach受け入れ境界

### TOMOS側で変更可能

- Agent-Reach診断結果を読むTOMOS側アダプター。
- Web、YouTube、GitHub、RSSの経路選択。
- Agent-Reach未導入時の既存TOMOS経路へのフォールバック。
- プラグイン一覧、設定画面、無効時表示。
- ローカル設定の保存と読み込み。
- 関連するTOMOS側テスト、ドキュメント、リリース手順。

### TOMOS側で変更しない

- Agent-Reach本体リポジトリ。
- Agent-Reachの依存関係、仮想環境、内部コマンド実装。
- 外部サービスのログイン、Cookie、SNS投稿、外部書き込み。
- APIキー、認証情報、秘密情報の保存形式。
- ユーザー確認なしの外部送信。

### 受け入れ条件

- Agent-Reach未導入環境でも既存TOMOS機能が動く。
- 外部アクセスは初期OFFまたは実行前確認がある。
- 失敗時は既存TOMOS経路へ一度だけ戻る。
- 取得結果はユーザー確認なしでMemoryへ保存しない。
- `PLUGIN.md` に必須10項目が定義されている。

## 記入例

```md
## 作業対象
- docsのみ

## 触らないもの
- Agent-Reach本体コード
- Internet Layer
- 外部API連携

## 検証
- Markdown構文確認
- git diff --check
```

## 変更時の確認方法

- `git status --short --branch` で既存変更を確認する。
- `git diff --check` で不要な空白や破損を確認する。
- コード変更がある場合は対象テストを実行する。
