# TOMOS AGENTS

## 目的

TOMOS本体をAIエージェントが安全に編集・調査・検証するための作業ルールを定義する。

## 対象範囲

- TOMOS本体のコード、ドキュメント、テスト、リリース手順
- ローカルAI、PWA、契約書管理、教材パック、プラグイン導線
- Agent-Reach本体コードは対象外

## 必須項目

- 作業前に対象範囲を確認する。
- 既存の未コミット変更を勝手に戻さない。
- 依存追加、削除、外部API実行、production変更は事前確認する。
- UI変更は `DESIGN.md` に従う。
- 会話・キャラクター変更は `VOICE.md` に従う。
- 記憶・保存・検索変更は `MEMORY.md` に従う。
- プラグイン仕様は `PLUGIN.md` に従う。
- 変更後は関連テスト、構文チェック、差分確認を行う。

## 任意項目

- 変更理由
- 影響範囲
- 手動確認手順
- ロールバック方法
- 未解決の確認事項

## 禁止事項

- 個人情報、APIキー、秘密情報をログやサンプルに書くこと。
- Agent-Reach本体、Internet Layer、Web/GitHub/YouTube/RSS連携処理をこの作業で変更すること。
- 既存UIや既存プラグイン仕様をテンプレート整備だけで変更すること。
- ユーザーの未コミット変更を無断で上書きすること。
- 本番データ、外部API、課金、認証設定へ無断で触ること。

## 記入例

```md
## 作業対象
- docsのみ

## 触らないもの
- Agent-Reach本体コード
- Internet Layer
- 外部API連携

## 検証
- Markdown構文確認
- git diff --check
```

## 変更時の確認方法

- `git status --short --branch` で既存変更を確認する。
- `git diff --check` で不要な空白や破損を確認する。
- コード変更がある場合は対象テストを実行する。
