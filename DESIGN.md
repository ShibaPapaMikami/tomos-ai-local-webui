# TOMOS DESIGN

## 目的

TOMOSのUIを、ローカルAIアプリとして一貫した見た目、操作感、情報密度に保つ。

## 対象範囲

- TOMOS本体の画面、設定、チャット、プラグイン導線、PWA
- 公式プラグインの標準UI
- 契約書管理、教材パック、マイキャラ、モデル設定などの管理画面

## 必須項目

- 主要操作は分かりやすい日本語で表記する。
- ボタン名は動詞で始める。
- 危険操作、未接続、未実装、テスト中は見た目でも分かるようにする。
- 画面右カラムは可能な限り広く使い、不要な入れ子カードを増やさない。
- 設定画面は右カラム内スクロールを基本にする。
- スマホ画面では下部入力欄と主要操作を優先する。
- 色、余白、角丸、枠線は既存TOMOS UIに合わせる。
- 文字は折り返し、はみ出し、重なりを避ける。

## Visual Theme

TOMOSはローカルAIの作業アプリである。見た目は落ち着いた、読みやすい、繰り返し使いやすい方向に寄せる。

- 情報密度は中程度から高め。
- 装飾より操作の分かりやすさを優先する。
- 契約書、教材、設定などの業務系画面は静かで実務的にする。
- キャラクター会話やスマホPWAでも、過度に派手な演出は避ける。

## Color Palette

- 背景: 明るい生成り系または白系を基本にする。
- 文字: 高コントラストの濃いグレーを基本にする。
- アクセント: 青または緑を主操作に限定して使う。
- 成功: 緑系。
- 警告: 黄または赤系。
- 無効: 薄い枠線、低いコントラスト、クリック不可の見た目。

## Typography

- 見出し: 短く、階層を増やしすぎない。
- 本文: 日本語で読みやすい行間を確保する。
- 補足: 小さくしてよいが、重要な注意を薄くしすぎない。
- ボタン: 1行で読める短い動詞句を優先する。
- コード、パス、ID: 等幅表示にする。

## Components

- 主ボタン: 画面の主操作にだけ使う。
- 補助ボタン: 枠線ボタンを基本にする。
- 無効ボタン: 押せない理由が分かる文言か補足を添える。
- タブ: 大きな機能切替に使う。
- 設定行: ラベル、入力、補足の順にする。
- カード: 個別アイテムや確認ブロックに限定する。
- 入れ子カード: 原則使わない。

## Layout

- 右カラム全体を作業領域として使う。
- 設定や管理画面は右カラム内スクロールを基本にする。
- 左メニュー表示中に不要なチャット上部情報を強調しない。
- スマホでは入力欄と主要アクションを下部に寄せる。
- 横並び操作は意味が近いものだけに限定する。

## Depth

- 影は控えめにする。
- 枠線と余白で階層を表現する。
- 画面全体をカードの中に閉じ込めない。
- 強い囲みは重要な確認、警告、入力ブロックに限定する。

## Do / Don't

Do:

- 「契約書を選択」「契約書を取り込み」のように具体的に書く。
- 読込中、完了、失敗を同じ場所に表示する。
- テスト中の機能は薄くして押せない状態にする。

Don't:

- 「Submit」「Execute」など英語の汎用表現を使わない。
- 似た強さのボタン色を乱用しない。
- 実在人物、実在企業、金額、秘密情報をUIサンプルに使わない。

## Responsive

- PC: 左メニューと右カラムで情報を整理する。
- タブレット: 左メニューを畳める前提にする。
- スマホ: ドロワー、下部入力、縦並び操作を基本にする。
- PWA: オフライン、未接続、PC接続中の状態を明示する。

## Agent Prompt Guide

AIエージェントがUIを作る場合は、以下を優先する。

- TOMOS既存UIに合わせる。
- 日本語の操作名を使う。
- 入れ子カードを避ける。
- 状態表示を必ず用意する。
- PCとスマホで文字がはみ出さないようにする。

## 任意項目

- 画面状態のスクリーンショット
- 空状態、読込中、エラー、成功時の表示案
- PC/スマホの表示差分
- アクセシビリティ上の注意

## 禁止事項

- 個人情報や実在ユーザー情報をUIサンプルに使わない。
- 装飾だけのカード、過剰なグラデーション、意味のない区切りを増やさない。
- 未実装機能を通常利用できるように見せない。
- 外部公開、クラウド同期、外部API連携を標準UIの前提にしない。
- Agent-ReachのUIやInternet Layer処理をこの仕様だけで変更しない。

## 記入例

```md
## 画面名
契約書管理

## 主操作
- 契約書を選択
- 契約書を取り込み
- 取り込み漏れチェック

## 状態表示
- 未選択: 契約書未選択
- 読込中: 契約書を読み取り中
- 完了: 契約書候補を作成しました
```

## 変更時の確認方法

- PC幅とスマホ幅で文字がはみ出さないか確認する。
- 無効ボタンが押せない見た目になっているか確認する。
- 既存のTOMOS配色、余白、ボタン形状から大きく外れていないか確認する。
