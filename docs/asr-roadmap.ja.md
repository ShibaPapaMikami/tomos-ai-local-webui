# 音声入力/ローカルASR ロードマップ

目的: チャットフォームに音声入力を追加し、認識結果を即送信せず、まず入力欄へ文字として入れる。

## 方針

- 既存のテキスト入力を壊さない。
- マイクボタンはフォーム内の小さなアイコンにする。
- 音声認識結果は送信前にユーザーが編集できるようにする。
- Mac/Windowsどちらでも導入しやすい構成にする。
- 重いASRモデルを必須にせず、軽量候補に切り替えられる余地を残す。
- Whisper単体をリアルタイム扱いしない。VAD、分割、常駐ワーカー、前後文脈を組み合わせて実用的な音声入力にする。

## 参考方針

- Qiita「ローカルで動く日本語STTを作る」連載:
  - Whisperは30秒チャンクの音声特徴量をテキストへ変換するモデルとして理解する。
  - リアルタイム化ではVADが重要になる。
  - ローカル候補は `faster-whisper`、`whisper.cpp`、日本語特化モデルを比較する。
- DevelopersIO「pipecatでSTT・LLM・TTSを組み合わせて音声AIボットを動かしてみる」:
  - 音声入力、STT、LLM、TTS、音声出力を1つの巨大処理にせず、差し替え可能なパイプラインとして扱う。
  - どこまでが1回の発話かをVADで判定する。
  - 今回はTTSや音声AIボット化までは対象外。採用するのは「パイプライン化」と「VADで発話区切りを扱う」考え方だけ。
  - 途中表示は「見た目のリアルタイム」と「確定文字起こし」を分ける。Whisperは停止後確定を基本にし、途中表示は分割音声の実験機能として扱う。

## 候補

### NVIDIA Nemotron 3.5 ASR Streaming 0.6B

- 用途: 高品質なローカル音声認識候補。
- 懸念: 学生PCでは重い可能性がある。
- 位置づけ: 精度重視の上位候補。最初の学生向け既定にはせず、後から選べるようにする。
- 検証項目:
  - Mac Apple Siliconで動くか
  - Windows GPUなしで現実的な速度か
  - 日本語の認識精度
  - 英語の認識精度
  - ストリーミング入力の遅延
  - インストールサイズ
  - モデルライセンスと再配布条件

### 軽量ASR候補

- 用途: 学生PC向けの低負荷モード。
- 条件:
  - ターミナル操作なしで導入できる
  - Mac/Windowsで同じUIから使える
  - 日本語の短文入力に十分使える
- 候補:
  - `whisper.cpp`: Mac/Windows両対応で、tiny/baseなど小さなモデルから試せる。最初の実装候補。
  - `faster-whisper`: CTranslate2ベース。GPUやApple Silicon環境で高速化しやすく、日本語特化モデルと組み合わせやすい。次の有力候補。
  - `kotoba-whisper`: 日本語精度重視候補。学生PCでの速度と導入の重さを検証してから採用判断する。
  - `Vosk`: 小型モデルが軽く、オフライン・日本語対応・pip導入がしやすい。低スペックPC候補。
  - `sherpa-onnx`: ONNX Runtimeベースで拡張性が高い。WebSocketサーバーや組み込み用途まで広げやすい。

### Pipecat型パイプライン候補

今回すぐPipecat本体を入れるわけではない。先にこのアプリ内で小さく同じ構造を作る。

```text
ブラウザ録音
  -> 音声バッファ
  -> VAD/無音判定
  -> ASRエンジン
  -> 暫定テキスト
  -> 入力欄
  -> 停止後の確定テキスト
```

将来的にTTSや音声対話へ広げる場合だけ、Pipecat本体の採用を再検討する。

### whisper.cppで確認できた実行候補

- `whisper-cli`: 現在使っている停止後の確定文字起こし。`--vad` はあるがVADモデルが別途必要。
- `whisper-stream`: マイク入力を直接扱うストリーミング候補。ブラウザ録音UIとどう接続するかは別途設計が必要。
- `whisper-server`: HTTPサーバー候補。`--vad` と `/inference` を使えるため、将来的な常駐Worker候補。
- `whisper-vad-speech-segments`: 音声ファイルから発話区間を抽出する候補。VADモデルの同梱またはDL管理が必要。

次の実装では、ブラウザ録音を保ったまま `whisper-server` を裏で常駐させる案と、先にブラウザ側の簡易RMS無音判定で分割する案を比較する。

## UI仕様

- 入力フォーム右側にマイクボタンを追加する。
- 録音中はマイクボタンを停止ボタンに変える。
- 認識中はフォーム近くに `音声認識中... 3秒` のように表示する。
- 認識結果は入力欄へ追記する。
- 認識結果を自動送信しない。
- 失敗時はチャットではなく入力欄付近に短く表示する。

## テスト

- マイクボタンが表示される
- 録音開始/停止の表示が切り替わる
- 認識結果が入力欄に入る
- 認識結果が自動送信されない
- ASR未導入時に分かりやすい案内が出る
- 音声入力なしでも通常チャットが壊れない
- `whisper.cpp tiny` は短い音声を高速に処理できる
- `whisper.cpp large/turbo` は精度が上がるが遅くなる
- `ローカル途中処理` を選ぶと、録音中に途中処理の表示が出る
- VADで無音区間を送らない
- 暫定テキストが確定テキストで重複しない
