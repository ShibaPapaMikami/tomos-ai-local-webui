# 言語モデル画面の整理

## 目的

学生や初心者がモデル名の違いで迷わず、用途だけで必要なAIを追加・選択できる画面にする。

## 通常表示

通常画面には、次のおすすめ3用途だけを表示する。

- 標準AI
- コード作業
- 高性能AI

各カードには用途、取得状態、操作ボタンに加えて、実際に使用する内部モデル名を小さく表示する。内部モデル名だけをまとめた折りたたみは削除する。

## 詳細モデル

「詳細モデルを表示」と「実験モデルを表示」は通常画面から削除する。おすすめより優先度が低いモデルを、新規ダウンロード候補やチャット欄の選択候補には出さない。

既存の取得済みモデルや保存済み設定は削除しない。互換性のため内部データには残す。

## 不要モデルの管理

おすすめ3用途に含まれないモデルがすでに取得済みの場合だけ、「不要なモデルを削除」を表示する。ここではモデル名、使用容量に関する注意、アンインストール操作だけを提供する。

自動アンインストールは行わず、ユーザー操作と確認を必須にする。

## 開閉状態

モデル取得状況の定期更新で画面を再描画しても、ユーザーが開いた折りたたみは閉じない。開閉状態を描画前に取得し、同じ識別子の要素へ復元する。

## チャット欄との整合

チャット欄には、自動（おすすめ）と、取得済みかつおすすめ3用途に該当するモデルだけを表示する。詳細・実験・非表示モデルは自動選択と手動選択の両方から除外する。

## テスト

- 定期再描画後も「不要なモデルを削除」の開閉状態が維持される
- おすすめカードに内部モデル名が表示される
- 内部モデル名専用・詳細・実験の折りたたみが表示されない
- 非推奨モデルは新規ダウンロード候補とチャット候補に入らない
- 取得済みの非推奨モデルだけ、削除画面に表示される
- 既存のモデル取得状態と保存キーを変更しない

## 対象外

- モデル本体の自動削除
- モデルの再ダウンロード
- サーバー側のモデル分類変更
- Enterpriseモデルの追加
