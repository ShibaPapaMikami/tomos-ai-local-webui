# 標準AI選択状態の保持 設計

## 目的

標準AIが未導入でも、利用者が選んだ「標準AI」を「自動（おすすめ）」へ戻さない。代わりに言語モデル画面を開き、Qwen3 4B Instruct 2507のダウンロードボタンを表示する。

## 動作

- モデル用途の選択状態を、実際のモデルIDとは別に保存する。
- 標準AIを選んだ時に利用可能な標準モデルがなければ、選択表示は標準AIのまま維持する。
- 同時に言語モデル画面を開き、既存の標準AIカードとダウンロードボタンを表示する。
- ダウンロード完了後は、保持中の標準AI用途がQwen3 4B Instruct 2507を自動的に利用する。
- Qwen 2.5 3Bは引き続き標準AIへ自動選択しない。

## 保存

- `gemma4.composerPurpose`へ`auto`、`standard`、`coding`、`high-performance`のいずれかを保存する。
- 既存利用者は、保存済みモデルIDから用途を導出して移行する。

## 検証

- 標準モデル未導入でも用途選択が`standard`のままになる。
- 標準モデル導入済みでは従来どおりQwen3 4Bを利用する。
- Qwen 2.5 3Bのみの環境では標準モデルとして選択されない。
- 既存のモデル選択・設定テストが壊れない。
