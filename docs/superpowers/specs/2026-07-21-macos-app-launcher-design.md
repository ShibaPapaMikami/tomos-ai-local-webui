# macOSアプリアイコン起動 設計

## 目的

TOMOS AIをmacOSの一般的なアプリと同じように、アプリケーション一覧とLaunchpadからアイコンで起動できるようにする。AI処理は引き続きローカルで行い、現在のブラウザーUIを維持する。

## 利用者の操作

- インストール後、`/Applications/TOMOS AI.app`をダブルクリックする。
- TOMOSのローカルサーバーが未起動なら起動し、利用可能になったら既定ブラウザーでTOMOSを開く。
- すでに起動している場合は重複起動せず、既定ブラウザーでTOMOSを開く。
- 起動できない場合は、原因と次の操作を日本語のダイアログで表示する。

## アプリ構成

- TOMOS本体を`TOMOS AI.app/Contents/Resources/Gemma4_12B`へ格納する。
- `TOMOS AI.app/Contents/MacOS/TOMOS AI`を起動専用ランチャーにする。
- ランチャーはアプリ内のTOMOS本体を作業フォルダーとして既存のMac起動処理を呼び出す。
- DockとLaunchpadにはTOMOSのアプリアイコンと`TOMOS AI`の名称を表示する。
- Ollamaのモデル、ブラウザー保存データ、長期記憶、教材パックなど、アプリ外に保存される既存データは変更しない。

## 既存インストールからの移行

- 現在の`/Applications/Gemma4_12B`は、新しいアプリ版のインストール時に自動削除しない。
- 新しい`TOMOS AI.app`が正常に起動し、既存データを確認できた後で、利用者が古い`/Applications/Gemma4_12B`を削除できるよう案内する。
- 旧フォルダーを残した状態でも、新しいアプリは必ずアプリ内の本体を使い、旧版を誤って起動しない。
- 移行確認前のデータ削除や設定初期化は行わない。

## 署名と公証

- `TOMOS AI.app`は個人名義の`Developer ID Application`で署名する。
- PKGは登録済みの`Developer ID Installer`で署名する。
- 公開前にApple公証、チケット添付、Gatekeeper検証を行う。
- 必要な証明書がない場合は、未署名アプリや未署名PKGを公開しない。
- 証明書、秘密鍵、App Store Connect APIキーはリポジトリへ保存しない。

## エラー処理

- サーバー起動のタイムアウト、必要コマンド不足、ポート競合を区別して案内する。
- エラー時に複数のサーバープロセスを残さない。
- 技術的な詳細はログへ残し、利用者向けダイアログには必要な操作だけを表示する。

## 検証

- 未起動状態からアイコンでサーバーとブラウザーが起動する。
- 起動済み状態でアイコンを押してもサーバーが重複しない。
- アプリケーション一覧、Launchpad、Dockに名称とアイコンが正しく表示される。
- 既存の設定、長期記憶、教材パック、取得済みモデルを維持できる。
- 新規Mac相当の環境と既存`Gemma4_12B`フォルダーがある環境の両方で動く。
- `codesign`、`pkgutil`、`stapler`、`spctl`の検証をすべて通す。
