## 1. コード理解 有効/無効

現状:

- プラグイン状態は `gemma4.plugins` に保存される。
- `コード理解` は管理画面のプラグイン一覧で追加/削除できる。内部データ名は `codegraph`。
- フォルダー側には `folder.plugins.codegraph.enabled` / `status` / `summary` が保存される。
- フォルダー編集画面では、コード理解が追加済みの場合だけ `コード理解` 行が表示される。
- フォルダー単位で有効/無効を切り替えられる。
- `準備する` で `/api/workspace/codegraph/init` を呼び、結果を該当フォルダーへ保存する。
- チャット送信時は、該当フォルダーの `codegraph.enabled && status === "ready"` の場合だけコード理解の要約を文脈へ入れる。

残り:

- ロードマップ上のチェック状態が実装状況とずれている。
- UI上で「有効にしただけ」と「準備済み」の差が少し分かりにくい。
- `.codegraph/` 削除や再作成の明示的な安全導線はまだない。
- 現在の実装はアプリ内簡易要約生成で、外部のコード解析バイナリ管理ではない。

判定:

- フォルダー単位有効/無効のMVPはかなり進んでいる。
- 次は新規機能を広げるより、状態表示と手動QAを固めるのが最小。

## 2. スマホPWA 単体保存

現状:

- PWA manifest、service worker、オフライン画面、スマホ画面はある。
- スマホ版は `gemma4.mobileChat` にチャット履歴を保存する。
- 旧 `gemma4.mobileNotes` からチャット形式へ移行する処理がある。
- `gemma4.mobilePcConnection` にPC接続先と6桁コードを保存する。
- `gemma4.mobileCharacterProfile` にPC側QRから受け取った簡易キャラ設定を保存する。
- スマホ単体AIはWASM実験と会話フォールバックまで入っている。

残り:

- 単体保存はまだチャット中心。
- 学習ノート、軽い記憶メモ、キャラクター設定をスマホ側で明示的に作成/編集するUIはまだ弱い。
- 送信データの種類も現状は `gemma4-mobile-chat` が中心で、学習ノート/記憶メモとして分離されていない。
- モデル本体やPCローカルパスを保存しない方針は守れているが、保存対象のUI整理が残る。

判定:

- 「外出先でPCなしでもチャットを残す」はできている。
- 「学習ノート/軽い記憶メモとして単体保存」は次の最小実装対象。

## 3. PC QR取り込み

現状:

- PC側「スマホ接続」画面にQR、6桁コード、有効期限、接続先候補が表示される。
- QRにはスマホURL、PC接続先、ペアリングコード、簡易キャラ設定が含まれる。
- スマホ側はQRパラメータから接続先とコードを保存する。
- スマホ側の `PCへ送信` は `/api/mobile/import-chat` へ送る。
- サーバー側はペアリングコードを検証し、受信データをメモリ上の取り込み待ちキューへ入れる。
- 重複送信はfingerprintで検出される。
- PC側は `スマホ受信を確認` で取り込み待ちを取得し、チャット履歴として保存できる。

残り:

- 取り込み待ちキューはメモリ上なので、PCサーバー再起動で消える。
- ペアリングコードは期限付きだが、取り込み成功後に即無効化する仕様は未実装。
- PC側の取り込み先は現状チャット履歴中心で、キャラクター記憶/学習セット/学習ノートへの振り分け確認は未実装。
- 接続済み端末一覧、端末名、最終接続時刻、端末解除は未実装。

判定:

- QR経由でスマホチャットをPCチャット履歴へ送る流れは動いている。
- 次は「PC側で取り込み内容を確認して、チャット/記憶/学習へ分ける」が最小価値。

## 4. キャラクター記憶 一覧/編集/削除

現状:

- `gemma4.character` にキャラクター設定を保存する。
- `gemma4.characterMemorySets` に記憶セットを保存する。
- マイキャラ画面に記憶タブがある。
- 記憶の一覧、カテゴリ表示、検索、手動追加、編集、削除がある。
- `覚えて` 系の発話から候補を作り、確認して保存する導線がある。
- `auto` モードでは安全そうな名前/好み/学習目標だけ自動保存する。
- センシティブ情報は記憶候補から除外する処理がある。

残り:

- スマホから送られた内容をキャラクター記憶としてPC側で取り込む導線がない。
- 記憶セットの複数キャラ運用やセット切替は深掘りできていない。
- 削除は即時で、確認ダイアログやUndoは記憶単体にはない。

判定:

- PC側の一覧/編集/削除MVPは実装済み。
- 優先度4として、今回は新規実装を広げない。

## 次に実装すべき最小タスク3つ

1. コード理解の状態表示とQAチェックを固める
   - フォルダー設定で「無効」「有効だが未準備」「準備中」「使用可能」「エラー」を明確に表示する。
   - Safariで、プラグイン追加、フォルダーで有効化、準備、チャット文脈反映まで確認する。
   - ロードマップのチェック状態を実装状況に合わせて更新する。

2. スマホPWAに単体保存の種別を追加する
   - スマホ内データを `chat` / `learning-note` / `character-memory` の種別で保存できるようにする。
   - 既存の `gemma4.mobileChat` は壊さず、互換移行する。
   - 送信前に未取り込み件数を種別別に見せる。

3. PC QR取り込みの確認画面を拡張する
   - 受信したスマホデータをPC側で一覧表示する。
   - 取り込み先を `チャット履歴` / `キャラクター記憶` / `学習セット` から選べるようにする。
   - 取り込み成功後は重複を避け、スマホ側にも取り込み済みとして残せる現在の流れを維持する。
