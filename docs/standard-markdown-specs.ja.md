# TOMOS Standard Markdown Specs

## 目的

TOMOS本体、公式プラグイン、教材パック、キャラクター体験を同じ前提で設計・確認できるように、5種類のMarkdownを標準仕様として扱う。

この仕様は人間とAIエージェントの共通仕様書であり、現時点ではアプリが直接読み込む設定ファイルではない。

## 対象ファイル

| ファイル | 役割 | TOMOS本体 | プラグイン |
| --- | --- | --- | --- |
| `AGENTS.md` | 実装ルール、作業境界、安全ルール、検証ルール | 必須 | 実装依頼がある場合は必須 |
| `DESIGN.md` | UIデザイン、色、余白、ボタン、カード、レスポンシブ | 必須 | 公式プラグインは必須 |
| `VOICE.md` | キャラクター、口調、敬語、呼び方、相槌、NG表現 | 必須 | 会話・音声・アバター系は必須 |
| `MEMORY.md` | 記憶、保存対象、保存禁止、忘れる条件、スコープ | 必須 | 記憶・教材・資料検索系は必須 |
| `PLUGIN.md` | プラグイン仕様、権限、設定、UI導線、参照範囲 | 必須 | 必須 |

## TOMOS本体用とプラグイン用の違い

TOMOS本体用は、アプリ全体の既定動作、標準UI、標準口調、標準記憶ルール、拡張機能の共通安全境界を定義する。

プラグイン用は、特定プラグインが追加する機能だけを定義する。TOMOS本体の標準仕様を上書きせず、足りない範囲だけを補足する。

## 共通必須項目

- 目的
- 対象範囲
- 必須項目
- 任意項目
- 禁止事項
- 記入例
- 変更時の確認方法

## DESIGN.md の標準セクション

`DESIGN.md` はAIエージェントがUIを再現しやすいように、最低限以下を含める。

- Visual Theme: TOMOSらしい雰囲気、密度、避ける表現
- Color Palette: 背景、文字、アクセント、警告、成功、無効状態
- Typography: 見出し、本文、補足、ボタン、モノスペースの使い分け
- Components: ボタン、入力欄、タブ、カード、一覧、設定行、モーダル
- Layout: 余白、グリッド、右カラム、スマホ下部入力、スクロール領域
- Depth: 影、枠線、面の重なりを増やしすぎないルール
- Do / Don't: 推奨と禁止の具体例
- Responsive: PC、タブレット、スマホ、PWAの表示差分
- Agent Prompt Guide: AIエージェントがUI作成時に参照する短い指示

## 共通禁止事項

- 個人情報、APIキー、秘密情報、社内未公開情報を書かない。
- 実在ユーザー、実在取引先、実在契約、未公開URLをサンプルにしない。
- 外部API実行を前提にしない。
- Agent-Reach本体コード、Internet Layer、Web/GitHub/YouTube/RSS連携処理へ介入しない。
- 既存プラグインの仕様をこのテンプレートだけで変更しない。
- 既存UIをこの作業で変更しない。

## Agent-Reachとの境界

この標準MarkdownはAgent-Reach実装と別作業である。`PLUGIN.md` はAgent-Reach担当が後で必要項目を埋められる汎用テンプレートとして扱う。

この作業では以下を行わない。

- Agent-Reach本体コードの変更
- Internet Layer実装
- Web/GitHub/YouTube/RSS連携処理
- プラグイン実行基盤の変更
- 依存追加
- 外部API実行

## 確認方法

- 5ファイルの役割が重複しすぎていないか確認する。
- TOMOS本体用とプラグイン用の違いが分かるか確認する。
- `PLUGIN.md` に最低限の必須項目が揃っているか確認する。
- `DESIGN.md` がTOMOSのUI一貫性を守る内容か確認する。
- `VOICE.md` が通常チャットとキャラクター再現性の両方に使えるか確認する。
- `MEMORY.md` が保存対象、保存禁止、忘れる条件、スコープを含むか確認する。
