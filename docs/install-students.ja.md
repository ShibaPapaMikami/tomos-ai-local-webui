# 学生向けインストール手順

専門学校や授業で配布するときは、GitHub Releases から OS に合ったインストーラーを配布します。古いバージョンも Release として残るため、授業当日に動く版へ戻せます。

## 事前に必要なもの

- Ollama
- Python 3.11 以降
- 空き容量: 最低 15GB 程度
- 初回モデル取得用のネットワーク

モデルは数GBあります。授業前に学校側で一度起動し、必要なモデルを取得済みにしておくと安全です。

## Mac

1. GitHub Releases から `TOMOS_AI-vX.X.X-mac.zip` をダウンロードします。
2. ZIP を展開します。
3. `Gemma4_12B_全部起動.command` をダブルクリックします。
4. macOS に止められた場合は、右クリックして `開く` を選びます。
5. ブラウザーで `http://127.0.0.1:54876/` が開いたら成功です。

Web UI だけを起動したい場合は `Gemma4_12B_Web.command` を使います。

## Windows

推奨は `.msi` です。

1. GitHub Releases から `TOMOS_AI-vX.X.X-windows.msi` をダウンロードします。
2. `.msi` をダブルクリックしてインストールします。
3. デスクトップの `TOMOS AI Web UI` を開きます。
4. または、スタートメニューから `TOMOS AI > TOMOS AI Web UI` を開きます。
5. ブラウザーで `http://127.0.0.1:54876/` が開いたら成功です。

MSI 版では、Program Files 内の `.bat` ファイルを探して起動する必要はありません。
デスクトップかスタートメニューのショートカットから起動してください。

Windows に止められた場合は、`詳細情報` から実行します。

ZIP 版は、MSI でうまく動かないときの予備です。

1. GitHub Releases から `TOMOS_AI-vX.X.X-windows.zip` をダウンロードします。
2. ZIP を右クリックして `すべて展開` します。
3. `Gemma4_12B_Web.bat` をダブルクリックします。

ZIP 版では `.bat` ファイルを直接使います。ComfyUI など周辺機能もまとめて起動したい場合だけ `Gemma4_12B_All_Start.bat` を使います。

## モデルの取得

アプリを開いたら、左下の `設定` から `言語モデル` を開きます。

通常画面ではモデル名ではなく、次の用途を選びます。

- `自動（おすすめ）`: 入力内容に合わせてTOMOSが選びます。
- `標準AI`: 通常会話、翻訳、資料検索、教材パックに使います。
- `コード作業`: プログラムの作成や修正に使います。専用AIが未取得でも標準AIで一度だけ続けます。
- `高性能AI`: 画像を読むときに使います。

実際に使う内部モデル名は、`内部モデル名を確認` を開いたときだけ表示します。標準AIはQwen3 4B Instruct 2507、コード作業はAgentic Coder v2、高性能AIはGemma 4 12BまたはMLX版です。

Qwen 2.5 3Bは旧環境からの移行確認用で、標準AIや高速会話には自動選択しません。Qwen3 4Bが未取得の場合は勝手に別モデルへ切り替えず、標準AIのダウンロード案内を表示します。

画像を読むには高性能AIが必要です。未取得の場合は、画像を標準AIへ送らず、追加の案内が表示されます。

取得済みになっていないモデルは、画面の案内に従って取得します。Enterpriseモデルや実験モデルは学生向けの通常画面には表示されません。

## 資料検索で使えるファイル

現在の標準経路は `TXT`、`Markdown`、`PDF` です。検索結果には原文の根拠を付け、SQLite FTSまたはn-gram検索を使います。

`CSV`、`Excel`、LanceDB、Knowledge Graphは今回の標準機能には含まれません。利用可能とは表示しません。

## ローカル処理と企業サーバー

通常はPC内のローカルAIで処理します。企業サーバーを使う接続は初期OFFで、管理者の許可がある場合だけ有効にします。企業資料であっても自動では送信しません。

接続を有効にする画面では、`ローカル処理` または `企業サーバー送信` のどちらかを必ず表示します。モデル名を隠しても、送信先は隠しません。

## 今後の比較候補

- 音声出力: Qwen3-TTS 0.6B Baseを第一実験候補とします。現在のWhisper／Nemotron音声入力は変更しません。
- コード作業: Ornith 9Bは実機比較候補です。今回は追加も自動選択もしません。
- 高性能AI: Gemma E4Bは既存Gemmaとの実機比較候補です。今回は標準追加しません。

## 古いバージョンへ戻す

GitHub Releases には過去の ZIP も残します。新しい版で不具合が出た場合は、前回授業で使えた Release の ZIP をダウンロードして使ってください。

## よくある困りごと

- 起動しない: Ollama と Python が入っているか確認します。
- 画面が開かない: `http://127.0.0.1:54876/` を手動で開きます。
- モデルが遅い: `モデル自動` または `高速` を選びます。
- 音声入力が使えない: `設定 > 音声入力` でマイクとモデルを確認します。
- PDFや画像文字起こしが弱い: `拡張 > プラグイン` で `画像文字読み取り（OCR）` を確認します。
