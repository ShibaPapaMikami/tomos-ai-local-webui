# Gemma4_12B リファクタリングロードマップ

目的: 既存の動作を変えずに、壊れやすい巨大ファイルを段階的に分け、今後の機能追加と修正を安全にする。

## 方針

- 1回の変更は小さくし、毎回 `node --check web/app.js` と `python3 -m py_compile server.py scripts/standardize_training_data.py` を通す。
- UIや挙動を変えるリファクタは避け、まずは定数・純粋関数・描画関数の順に分離する。
- 既存のローカル保存データ、フォルダー、チャット、学習セットの形式は維持する。
- 手動確認は `docs/manual-test-checklist.ja.md` を使い、変更範囲に近い項目を優先する。

```text
コード理解

```

導入後、フォルダー編集に以下を表示する。

```text
[ ] コード理解を使う
このフォルダー内のコードのつながりを調べ、説明や修正をしやすくします。
```

方針:

- [ ] ONにしたフォルダーだけ CodeGraph を使う
- [ ] 全フォルダーで一律ONにはしない
- [ ] UI文言は「追加」「削除」「更新」「このフォルダーで使う」「準備中」「使用できます」「準備できませんでした。もう一度準備」に寄せる
- [ ] `.codegraph/` は対象フォルダー内に作られる前提にする
- [ ] `.codegraph/` の削除は破壊的操作に近いため、削除実装時は確認を必須にする

CodeGraph導入フロー:

1. 左下メニューから「プラグイン」を開く。
2. 「コード理解」の「追加」を押す。
3. インストール完了。
4. フォルダー編集を開く。
5. 「コード理解を使う」をONにする。
6. アプリが対象フォルダーを準備する。
7. チャットでコード説明や修正相談に使える。
