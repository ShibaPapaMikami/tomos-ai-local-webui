# 人物・関係メモアプリ Roadmap

## 位置づけ

人物・関係メモは、TOMOSの `アプリ` として扱う。
プラグインではない。

理由:

- 人物情報はチャット、Discord、LINE、Slack、メールなど複数機能から参照される共通データである
- プラグインごとに人物データを持つと、情報が重複し、関係メモが壊れやすい
- Local Context Coreの `person` EntityとRelationに載せることで、将来のKnowledge Graphに接続できる

## 対象カテゴリ

- 友達
- 恋愛
- 家族
- 仕事

## 登録項目

- 自分の情報
- 姓
- 名
- 表示名
- 呼び名
- 写真
- 関係カテゴリ
- 詳細関係
- 生年月日
- 性別
- 血液型
- MBTI
- 自分との関係メモ
- 自分との関係図
- 最終更新日

MBTIは任意入力の参考メモとして扱う。
公式診断として提供しない。
年齢は保存値ではなく、生年月日から表示用に計算する。
好きなこと、苦手なこと、会話の注意点は `自分との関係メモ` に集約する。

## 主な導線

### 人物登録

1. `人物・関係メモ` を開く
2. 自分の情報を登録する
3. 人物を追加する
4. 姓名、表示名、関係カテゴリ、詳細関係、写真、メモを登録する
5. 必要ならMBTI、血液型、生年月日、性別を登録する

### 相性メモ

1. 登録人物を開く
2. 自分との関係カテゴリ、詳細関係、メモを見る
3. 会話のコツ、注意点、避けた方がよい表現を表示する

相性メモは断定ではなく参考として表示する。

### 関係図

1. 自分の情報を中心に表示する
2. 登録人物を関係カテゴリ別に一覧表示する
3. 詳細関係をラベルとして表示する

### 返信支援

1. チャットで `送り先` を選ぶ
2. TOMOSが人物メモ、詳細関係、MBTI参考を参照する
3. その人に合わせた返信案を作る
4. 実際の送信はユーザーが確認して行う

## プラグインとの関係

人物・関係メモはアプリ本体側に置く。
外部連携はプラグイン側に分ける。

参照するプラグイン候補:

- Discord
- LINE
- Slack
- メール
- カレンダー

プラグインは人物データを持たず、人物IDまたは検索結果だけを参照する。

## Local Context Core連携

人物は `ContextEntity` として扱う。

```text
entityType: person
name
aliases
photoRef
age
gender
bloodType
personalityType
personalityTypeSource
notes
scope
```

関係は `ContextRelation` として扱う。

```text
fromEntityId -> relationType -> toEntityId
```

Relation type候補:

- `friend`
- `romantic_interest`
- `family`
- `work_contact`

MBTIの根拠:

- `self_reported`
- `user_reported`
- `estimated`
- `unknown`

## MVP

- [ ] 写真登録

- [ ] MBTIの任意登録

## やらないこと

- 外部SNSへの自動送信
- 公式診断
- 他人の属性の自動断定保存
- 採用、評価、医療、法的判断への利用
- クラウド同期前提の実装

## エンジニアへの指示

- [ ] まずUIは人物一覧、詳細、編集だけに絞る
- [ ] チャット返信支援は、送り先選択とプロンプト文脈追加までに絞る
- [ ] Discord/LINE/Slack/メール連携は後段のプラグインタスクに分ける
- [ ] MBTIは断定表示せず、任意メモとして扱う
- [ ] 自動保存候補は必ずユーザー確認を挟む
- [ ] Context Core実装前でも、将来移行できるように `personId`、`relationType`、`scope` を持たせる

## 受け入れ条件

- [ ] 人物・関係メモがアプリ枠として説明されている
- [ ] 友達、恋愛、家族、仕事を登録できる
- [ ] チャット返信支援で送り先を選べる
- [ ] プラグインは人物データ参照側として整理されている
- [ ] MBTIが公式診断や断定表現になっていない
- [ ] Local Context Coreとの接続方針が明記されている
