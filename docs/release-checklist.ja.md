# リリース前チェックリスト

学生がMac/Windowsで導入しやすい状態かを確認するためのチェックリストです。

## 起動

- [ ] Macで `Gemma4_12B_全部起動.command` が起動する
- [ ] Macで `Gemma4_12B_Web.command` が起動する
- [ ] Windowsで `Gemma4_12B_All_Start.bat` が起動する
- [ ] Windowsで `Gemma4_12B_Web.bat` が起動する
- [ ] 起動後、ブラウザーで `http://127.0.0.1:54876/` が開く
- [ ] 設定画面にアプリ版、モデル状態、利用モデルが表示される

## GitHub Release配布

- [ ] `bash scripts/make-release-archives.sh` でMac/Windows ZIPを作成できる
- [ ] `dist/Gemma4_12B-vX.X.X-mac.zip` が作成される
- [ ] `dist/Gemma4_12B-vX.X.X-windows.zip` が作成される
- [ ] ZIPに `.git/`、`ComfyUI/`、`.venv/`、`.venv-asr/`、モデルファイルが含まれていない
- [ ] GitHub Releaseに2つのZIPを添付している
- [ ] 古いReleaseを削除せず残している
- [ ] 学生向け案内に、使うReleaseのURLとOS別ZIP名を記載している

## モデル

- [ ] Gemma 4 12Bが未取得の場合、設定からダウンロード案内が分かる
- [ ] Gemma 4 Coder 12B Q4が未取得の場合、設定から状態が分かる
- [ ] 通常チャット、コード生成、翻訳の推奨モデルが学生にも分かる名前で表示される
- [ ] モデル自動、速度、精度の切り替えがチャットフォームから使える

## 基本チャット

- [ ] `おはよ` など短い会話で自然に短く返る
- [ ] `英訳して` / `日本語にして` で全文がチャットに返る
- [ ] 回答下に所要時間、モデル、用途、モードが表示される
- [ ] ユーザー発話とGemma回答のコピーボタンが機能する

## Web検索・天気

- [ ] Web検索ONで外部リンクに↗アイコンが出る
- [ ] Web検索ONで新しい情報を検索結果付きで答える
- [ ] `新潟の天気は？` でOpen-Meteoの結果が表示される
- [ ] 現在地を保存した後、`今日の天気は？` でブラウザ現在地由来だと分かる
- [ ] 現在地を使わない場合は既定地または地名指定で動作する

## フォルダー操作

- [ ] フォルダーを作成し、チャットがフォルダー配下に表示される
- [ ] フォルダー編集からローカルフォルダーを指定できる
- [ ] ローカルフォルダーのファイル一覧が表示される
- [ ] `helloと記載されたtextを保存して` のような単純保存が成功する
- [ ] 簡単なWebプログラム作成で `index.html` が保存され、プレビューできる

## 学習セット

- [ ] フォルダー編集で、そのフォルダーに使う学習セットを選べる
- [ ] 回答下の「修正して学習」から保存先学習セットを選べる
- [ ] 学習セットの中身を確認できる
- [ ] 保存済みの正しい回答を編集できる
- [ ] 学習用ファイルを書き出した後、その時点ではモデル自体が変わらないことが分かる

## 画像

- [ ] 画像をフォームの＋から添付できる
- [ ] 画像をクリップボードから1回だけ貼り付けられる
- [ ] ComfyUIが起動済みならチャットから画像生成できる
- [ ] 画像生成後にPCが重くなりすぎない

## 開発者確認

- [ ] JSヘルパーテスト一式が通る
- [ ] Python構文チェックが通る
- [ ] Safariで手動確認できる
- [ ] README.ja.md / README.en.md の手順が現在の起動ファイルと一致している
